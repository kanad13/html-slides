# Slide Deck Viewer

AI agents make it increasingly easy to create quick presentations as standalone HTML files. That is useful because HTML gives you direct control over layout, spacing, typography, animation, charts, images, and custom styling without fighting a traditional slide editor.

The missing piece is presentation. Once you have a folder full of HTML slide files, you need a reliable way to open them as a real deck: next and previous controls, Zen presentation mode, overview, speaker notes, a presenter view, keyboard shortcuts, and a clean audience display.

That is what `viewer.html` provides.

Open the viewer, point it at your HTML slide files, and present. No server. No Python. No Node. No build step. No external dependency.

## Who This Is For

Use this viewer when you:

- Ask an AI agent to mock up a presentation as separate HTML files.
- Generate slides programmatically and want to review or present them immediately.
- Prefer HTML/CSS control over rigid slide-editor layouts.
- Need to present locally without uploading private material.
- Want a portable viewer that can travel with the generated deck.
- Want presentation controls without adopting a full slide framework.

## The Basic Workflow

1. Create or generate one HTML file per slide.
2. Put those files in a folder.
3. Open `viewer.html`.
4. Choose the folder.
5. Present.

Example:

```text
my-presentation/
  viewer.html
  slides/
    slide01.html
    slide02.html
    slide03.html
```

You open `viewer.html`, choose the `slides/` folder, and the viewer builds the presentation.

## Why This Exists

HTML slides are powerful, especially when generated by AI agents:

- You can control the exact layout instead of working around slide templates.
- You can use CSS for responsive positioning, typography, grids, and visual polish.
- You can include charts, diagrams, SVG, canvas, and small interactions.
- You can keep every slide as a normal file that is easy to inspect and edit.
- You can generate many variants quickly.

But a folder of HTML files is not a presentation by itself. Manually opening files one by one is clumsy. Browser tabs do not give you slide order, progress, notes, overview, or presenter controls.

`viewer.html` is the bridge between "AI generated these HTML slides" and "I can present this deck now."

## Quick Start

1. Open `viewer.html` in Chrome, Edge, Brave, or Safari.
2. Click **Choose Folder**.
3. Select the folder containing your slide HTML files.
4. Use **Next**, **Previous**, arrow keys, or Space to present.
5. Press **Zen**, `Z`, or `F` to present fullscreen without viewer controls.

If folder picking is unavailable in your browser, click **Choose Files** and select the slide files manually.

## Recommended Slide Format

Use one standalone HTML file per slide:

```text
slide01.html
slide02.html
slide03.html
```

Each slide should include its own HTML and CSS. The viewer loads slides in iframes, so each slide is isolated from the viewer and from other slides.

Minimal example:

```html
<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="utf-8">
<meta name="viewport" content="width=device-width, initial-scale=1">
<title>Project Update</title>
<style>
  html, body {
    height: 100%;
    margin: 0;
    font-family: system-ui, sans-serif;
  }
  body {
    display: grid;
    place-items: center;
    padding: 6vw;
  }
</style>
</head>
<body>
  <main>
    <h1>Project Update</h1>
    <p>What changed, what shipped, and what is next.</p>
  </main>

  <aside class="notes">
    Mention the customer feedback before showing the roadmap.
  </aside>
</body>
</html>
```

The viewer uses the slide's `<title>` for the slide list and overview labels. If there is no title, it falls back to the first heading or the filename.

## Speaker Notes

Add private presenter notes inside a slide with:

```html
<aside class="notes">
  Talking points only the presenter should see.
</aside>
```

Press `N` to show the docked notes panel. Press `P` to open a separate presenter window with notes and a timer.

## Browser Support

Best experience:

- Chrome
- Edge
- Brave
- Safari

Folder picking uses browser-native local file selection. Firefox does not currently support the same folder picker, so use **Choose Files** there.

Drag-and-drop folder loading also depends on browser support. If dropping a folder does not work, use **Choose Folder**.

## Privacy and Offline Use

The viewer does not upload your slides. The selected files stay local to your browser tab.

When you choose a folder, the browser grants temporary access to those files. The viewer creates local object URLs so the slides can appear inside iframes. Closing the tab ends that access.

The viewer may remember lightweight preferences such as theme and last slide number in `localStorage`, but it does not store slide contents.

## Maintenance and Guardrails

Development rules live in [`AGENTS.md`](AGENTS.md). That file is the contract for future AI agents and maintainers.

The most important rule: normal use of `viewer.html` must not require external tools. Do not add Python, Node, npm, a server, a build step, CDN scripts, remote assets, analytics, or runtime dependencies.

If you change viewer behavior, update this README and keep `AGENTS.md` accurate.

## Why The Folder Picker Is Necessary

A static HTML page cannot scan arbitrary folders on your computer by path. Browsers block that for security reasons.

The folder picker is the browser-approved way for you to grant access to a specific local folder. That is why the viewer asks you to choose a folder instead of hardcoding a path.

## What This Is Not

This is not a slide authoring tool. It does not design slides, convert PDFs, sync audience devices, or generate slides from markdown.

It is a lightweight presentation viewer for HTML slide files you create elsewhere, often with help from AI agents.
