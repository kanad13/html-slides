Viewer contract

The viewer presents files; it does not become the slide system.

The runtime stays theme-agnostic, local-first, and dependent only on browser-native APIs.

Sandboxed iframesSlides render separately so one file cannot take over the viewer chrome.
Metadata extractionTitles and notes are read without executing slide scripts.
Local object URLsSelected slides and deck-local assets are represented in memory and revoked when replaced.
Call out that the viewer chrome theme does not alter slide contents. The slide owns its own visual system.
