Output rule

Every slide carries what it needs.

The viewer should not need a build step, shared stylesheet, package manager, or server to display a normal generated deck.

  1. 01Complete documentDOCTYPE, language, viewport, title, style, markup, and notes all live in the file.
  2. 02Inline tokensResolved token values are embedded inside each slide by default.
  3. 03Inline CSSLayout CSS consumes tokens instead of ad hoc colors.
  4. 04Local assets onlyOptional assets stay beside the deck and are selected by the user.
Stress that standalone means operational portability. It does not mean every deck must avoid local assets, but the default should be fully embedded HTML and CSS.
