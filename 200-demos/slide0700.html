Maintenance context

Future updates need the decisions behind the deck.

The sidecar records narrative, theme, assumptions, and unresolved questions so the next edit does not start from guesswork.

deck-context.md should preserve

  • purpose, audience, source material, and assumptions
  • deck arc, section flow, and slide map
  • theme path, source colors, token rationale, and readability adjustments
  • update notes, tradeoffs, and unresolved questions
This is the maintenance bridge. Without context, later changes tend to drift away from the original message and theme decisions.
