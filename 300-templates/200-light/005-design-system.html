Light Theme · Design System v2

Token-based slides with stable layouts and flexible branding.

This reference defines how colour, typography, surfaces, semantic states, and chart palettes should be used in light-theme presentation templates.

Token Hierarchy

Use roles, not raw colours.

Templates should consume token roles. This makes the same layouts work across default, Google-inspired, OpenAI-inspired, Microsoft-inspired, or client-specific brand systems.

Neutral Tokens

Used for surfaces, text, borders, structure, and readability.

Brand Tokens

Used for identity, accents, emphasis, and selected highlights.

Semantic Tokens

Used when colour communicates meaning such as success or risk.

Chart Tokens

Used only for charts, data visualisation, and legends.

Brand Adaptation

Change token values, not layouts.

Google-inspired

Primary blue, secondary red, tertiary yellow, quaternary green.

OpenAI-inspired

Green-led brand palette with neutral supporting tones.

Microsoft-inspired

Four-colour brand system mapped into brand tokens.

Most Important Rule

Layouts should never change when adapting to a new brand.

Only token values should change. This keeps the system scalable, predictable, and suitable for AI-generated decks across many brands.
